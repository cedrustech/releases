Latest Base Version - v5.0.24.9

New Features and Enhancements

New 1. Automation Added for Serial Numbers on Invoice Reports

»» Implemented automation to assign a unique serial number to each invoice record in the reports, improving tracking and organization.

New 2. Record Ownership and Modification History Tracking

»» Implemented a feature to display the Created By and Last Modified By history for each record, along with the current record owner. This functionality enables verification of ownership to ensure the record owner is a member of organization. If the owner is external, we now have the capability to make necessary changes. This feature is available for all objects and records within the application.

New 3. Automation for Default Record Ownership Assignment

»» Implemented automation to set default record ownership assignments, streamlining the process for new records within the application.

What's Changed/Fixed

Changed 1. Updated App Logo and Theme Colors

»» Replaced the current logo with the new logo for MME App. Also updated the brand and global header background colors to enhance the App’s visual identity.

Fixed 2. Issue with Changing Record Owner of Consignment

»» Resolved an error when attempting to change the Record Owner in the Consignment. This issue affects certain records, while others update successfully.

What's Changed/Fixed

Changed 1. Simplified and Reduced Mode of Transaction Options

»» In the Invoice section, removed one of the existing mode of transaction values, retaining only Cash and Card/UPI as the available options.

Fixed 2. Error Occurred While Changing Invoice Status to Paid

»» Resolved the newly occurred System.DmlException error that occurred while changing the invoice status to Paid.

Fixed 3. Cancelled Invoices Not Counted in Summary

»» Resolved the issue where cancelled invoices were not reflected in the Summary records. The total invoice count now accurately excludes cancelled invoices, and the Invoice Range field is updated accordingly.

Fixed 4. Cancelled Button Was Incorrectly Displayed on Already Cancelled Invoices

»» Resolved the issue where the Cancel button was incorrectly displayed on invoices that were already cancelled. The button will no longer appear for invoices in the cancelled status.

Fixed 5. Invoice Status Check Failure in Summary Month Removal

»» Corrected the functionality that allowed users to modify Summary Month field even when the invoice status was Paid or Cancelled. Users are now restricted from modifying Summary Month field under these conditions.

Fixed 6. Summary Month Field Modification Restricted

»» Users are now restricted from modifying the Summary Month field, as the system automatically selects the Summary Month record based on the chosen date.

Fixed 7. Summary Month Field Modification Restricted

»» Added restriction to prevent users from adding sale items once an invoice is closed.

Fixed 8. Summary Includes Pending Invoices

»» Resolved the issue where the Summary included details of both closed and pending invoices. Now, the Summary will only reflect information for closed invoices, ensuring accurate reporting.

Fixed 9. Dashboard Charts Include Pending Invoice Amounts

»» Resolved the issue where the Sales charts in Dashboard included amounts for pending invoices. The charts now accurately reflect only the amounts for closed invoices.

Changed 10. Increased Character Limit for Customer Name Field in Invoice

»» The character limit for the Customer Name field in the Invoice has been increased from 25 to 100 characters, allowing for more comprehensive entries.

Changed 11. Updated Transaction Icons for Both Payment Options

»» Updated the transaction icons to include a new icon for Cashless Payment (Card/UPI) as well as an icon for Cash Payment, enhancing visual clarity for both options.

New Features and Enhancements

New 1. Sales Performance Analyzer: Comprehensive Monthly Analysis

»» Implemented a Sales Performance Analyzer feature in the MME Project. This tool enhances the tracking and comparison of monthly sales data, providing users with comprehensive insights to effectively analyze sales performance.

1.1 Setup Relationship of New Summary Object with Invoice

»» A relationship field has been setup on the Invoice, linking it to the Summary custom object. When an invoice date is selected, the system automatically determines and selects the corresponding Month record in the field based on the chosen date.

1.2 Added Automation for Total Sale Amount and Sold Qty Count

»» Added automation to calculate and update the Total Sale Amount and Sold Quantity Count for each month in the Summary custom object whenever an invoice status is marked as Paid.

1.3 Status Indicator for Monthly Sales Summary

»» A status indicator has been implemented in the Summary object to visually represent each month's sales performance. This indicator uses symbols to compare the current month's sales to the previous month's performance.

1.4 Summary Creation Restriction Based on Existing Invoices

»» Implemented a restriction on creating new Summary records. The system will prevent the creation of a new Summary record if any invoices already exist in the database.

1.5 Automation to Update Previous Summary Status

»» Added automation to update the status of the previous Summary record to Completed whenever a new Summary record is created.

1.6 User Modification Restriction on Summary Record

»» Users are restricted from modifying the Summary record if its status is marked as Completed. Access to edit these records is no longer allowed.

1.7 Automation for Total Cash and Debit Card/UPI Amount

»» Implemented automation to calculate and update the Total Cash Amount and Debit Card/UPI Amount for each month in the Summary custom object whenever an invoice status is marked as Paid.

New Features and Enhancements

New 1. Introduced GST Fields in the Stock Object

»» Added new fields to the Stock - Now you can enter the HSN Code and the GST Rate for each product. This will help you keep track of taxes more easily.

New 2. Sale Object Update: Added Tax Information

»» Introduced new fields in the Sale object. You can now track Taxable Amount, CGST, and SGST for each sale. This enhancement will help you manage tax calculations more accurately.

New 3. Added New GST Value (3%) in Sale GST Field

»» Added a new GST value of 3% to the Sale GST field. You can now input this value in the GST field of the Sale object, making it easier to manage your GST calculations.

New 4. Updated Sales and Stock Reports with New GST Fields

»» We’ve enhanced the Sales and Stock reports to include the new GST fields. You can now view and analyze tax and GST-related information more effectively.

What's Changed/Fixed

Fixed 1. Sales Qty Rounding Decimals Issue

»» We have resolved the issue where the system rounded decimal quantities to the nearest integer during sales entry. Users will now see the precise decimal quantities entered, eliminating any confusion while maintaining accurate calculations on invoices.

Fixed 2. Improved Stock Tracking: Entire Year Quantities Available

»» The Stock section previously displayed quantities for the current month only. This issue has been fixed, and the component now shows quantities for the entire financial year.

New Features and Enhancements

New 1. Implemented a Functionality for Custom Sales Analytics Detail

»» The Sales Analytics feature now enables users to view detailed monthly sales data for each stock item, allowing for accurate cross-checking and verification of sales figures.

New 2. Implemented Invoice Cancellation Functionality

»» The Invoice Cancellation functionality has been implemented, allowing users to easily cancel the current invoice with the click of a button.

New 3. Added Sale History for Stock Items

»» The Sale History feature has been added, providing users with the option to view the total items sold for each stock item on a monthly basis.

New 4. Added Consignment Name in Contact Object

»» A new Consignment Name field has been added to the Contact object. This field automatically records the name of the consignment whenever a user or contact person is linked, making it easier to track related information.

New 5. Unique Contact Ownership Enforcement for Consignments

»» The Unique Contact Ownership Enforcement feature has been developed to ensure that each consignment has only one designated contact person. This feature prevents a contact record from being assigned as a consignment owner if they are already linked to another consignment, ensuring clear ownership and avoiding duplication.

What's Changed/Fixed

Fixed 1. Restricted Date Fields on Stock - Not Accept the Past 1 Year Later Date

»» The date fields on the Stock custom object have been updated to include a validation rule that prevents users from entering dates that are more than one year in the past. This ensures that only relevant and up-to-date dates can be recorded.

Fixed 2. Gatepass Number Validation Issue

»» The Gatepass Number validation issue has been resolved. The validation rule has been updated to accept letters, numbers, and hyphen symbols. The required format for the Gatepass Number is now GP-0000, ensuring proper entry without errors.

Changed 3. Added Consignment Name in Gatepass

»» The Consignment Name field has been added to the Gatepass, and the Contact Name lookup field has been removed. This change streamlines the information captured in the Gatepass, allowing for easier tracking of consignments.

Fixed 4. Removed Unnecessary Fields and Sections from Stock (New Entry)

»» The issue of unnecessary fields and sections appearing when creating a new stock entry has been addressed. This change reduces clutter and confusion in the interface, providing a cleaner and more streamlined experience for users.

Changed 5. Added New Picklist Value to Gatepass Category

»» A new value, Return of Goods has been added to the Gatepass Category field, providing users with an additional option for categorizing gatepass records.

New Features and Enhancements

New 1. Implemented GST Functionality for Invoices

»» Now, you can easily apply GST rates of 5%, 12% or 18% to your invoices, and the corresponding GST amount will be automatically calculated and added to the total amount.

New 2. Icon for Mode of Transactions

»» We've enhanced the user experience by adding icons for each mode of transaction. When you select a mode of transaction, such as "cash," you will see a Cash icon, making it even more intuitive to identify your chosen mode.

New 3. Total Amount Calculation in MoT Fields

»» Introducing automation that dynamically reflects the total amount in a specific field based on the selected mode of transaction. For example, if you choose the mode of transaction as "Cash," the total amount will be displayed in the Cash Amount field, while all other fields will show zero. This feature simplifies reporting by allowing you to segregate billing calculations based on the mode of transaction.

Fixed 2. Resolved Stock Name Error (STRING_TOO_LONG)

Fixed 6. Fixed Quick Actions Visibility Issue
